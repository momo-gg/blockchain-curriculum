# Appendix A: Glossary

A comprehensive glossary of blockchain and Solana terminology.

---

## Core Blockchain Concepts

### Account
An addressable storage unit on Solana that can hold data and SOL. Every piece of state on Solana lives in an account. Accounts have an owner (a program that controls them), a balance (lamports), and optional data.

### Address
A 32-byte public key encoded in Base58. Used to identify accounts, programs, and wallets on the network. Example: `11111111111111111111111111111111`

### Block
A collection of transactions processed together and added to the blockchain. On Solana, block timing is derived from slot timing and can vary (as of `solana-labs/solana` clock constants).

### Blockchain
A distributed, immutable ledger of transactions maintained by a network of nodes through consensus. Each block references the previous block, creating a chain.

### Commitment
The level of finality for a transaction or account query. Levels (from least to most final):
- **processed**: Seen by the connected node
- **confirmed**: Voted on by supermajority of stake
- **finalized**: Cannot be rolled back (see Solana commitment docs)

### Consensus
The mechanism by which network participants agree on the state of the blockchain. Solana uses a combination of Proof of Stake and Proof of History.

### Decentralization
Distribution of control across multiple parties without a central authority. Measured by number of validators, stake distribution, and geographic spread.

### Fork
When the blockchain diverges into two or more potential paths. Solana's consensus resolves forks to maintain a single canonical chain.

### Finality
The guarantee that a transaction cannot be reversed. On Solana, finality is expressed via commitment levels (as of [docs.solanalabs.com/consensus/commitments](https://docs.solanalabs.com/consensus/commitments)).

### Hash
A fixed-size output from a cryptographic function. Used for block identification, transaction signatures, and data integrity verification.

### Immutability
The property that once data is written to the blockchain, it cannot be changed or deleted.

### Lamport
The smallest unit of SOL. 1 SOL = 1,000,000,000 (10^9) lamports. Named after computer scientist Leslie Lamport.

### Ledger
The complete history of all transactions on the blockchain, maintained by validators.

### Node
A computer running Solana software that participates in the network. Can be a validator (producing blocks) or RPC node (serving queries).

### Slot
A time window (target ~400ms) during which a leader can produce a block. The fundamental time unit in Solana (as of `solana-labs/solana` clock constants).

### Transaction
A signed message instructing the network to execute one or more operations. Contains instructions, signatures, and account references.

## Solana-Specific Terms

### Anchor
The dominant framework for Solana program development. Provides Rust macros and conventions that reduce boilerplate and improve security.

### BPF (Berkeley Packet Filter)
The bytecode format Solana programs compile to. Programs are deployed as BPF binaries executed by the Solana runtime.

### Cluster
A set of validators working together to maintain the ledger. Main clusters:
- **mainnet-beta**: Production network
- **devnet**: Development and testing
- **testnet**: Validator testing

### Compute Units (CU)
A measure of computational resources consumed by a transaction. Limits vary by cluster and release (as of `solana-labs/solana`).

### Cross-Program Invocation (CPI)
When one program calls another program within a transaction. Enables composability between programs.

### Epoch
A period during which the leader schedule remains fixed. Length is defined by protocol constants and can change by configuration (as of `solana-labs/solana` clock constants).

### Geyser
A plugin system for validators that streams account and transaction data in real-time. Used for building indexers and analytics.

### IDL (Interface Definition Language)
A JSON file describing a program's instructions, accounts, and types. Generated by Anchor and used by clients for type-safe interactions.

### Leader
The validator responsible for producing blocks during a specific slot. Selected based on stake weight.

### Leader Schedule
A predetermined order of which validators will be leaders for upcoming slots. Generated for each epoch based on stake distribution.

### Priority Fee
An optional additional fee paid to incentivize validators to include a transaction, especially during congestion. Measured in micro-lamports per compute unit.

### Proof of History (PoH)
Solana's innovation for proving time passage without coordination. A verifiable delay function that creates a historical record.

### Proof of Stake (PoS)
Consensus mechanism where validators stake SOL to participate in block production. More stake = higher probability of being selected as leader.

### Program
Executable code deployed on Solana (analogous to "smart contract" on other chains). Programs are stateless and process instructions.

### Program Derived Address (PDA)
A special type of address derived from a program ID and seeds that doesn't have a corresponding private key. Used for program-controlled accounts.

### Rent
The SOL cost for storing data on-chain. Accounts must maintain a minimum rent-exempt balance (as of [solana.com/docs](https://solana.com/docs)).

### Rent Exemption
When an account holds enough SOL to cover rent indefinitely (minimum balance varies by data size, refunded on close).

### RPC (Remote Procedure Call)
The API interface for interacting with Solana nodes. Used to query state, submit transactions, and subscribe to updates.

### Sealevel
Solana's parallel smart contract runtime that enables concurrent transaction processing on multiple cores.

### Slot
The period (target ~400ms) in which a validator can produce a block. Multiple slots may pass between blocks (as of `solana-labs/solana` clock constants).

### SOL
The native cryptocurrency of Solana, used for transaction fees, rent, and staking.

### Stake
SOL delegated to a validator to participate in consensus. Stakers earn rewards proportional to their stake.

### System Program
The native Solana program that handles basic operations: creating accounts, transferring SOL, allocating space.

### Token Program
The SPL program that implements fungible token functionality: minting, transferring, burning tokens.

### Tower BFT
Solana's consensus algorithm, a PoH-optimized version of PBFT (Practical Byzantine Fault Tolerance).

### Turbine
Solana's block propagation protocol that breaks blocks into smaller packets for efficient distribution across the network.

### Validator
A node that participates in consensus by voting on blocks and (if selected as leader) producing new blocks.

### Vote Account
An account associated with a validator that tracks their voting history and stake.

## Development Terms

### ATA (Associated Token Account)
A deterministically derived token account address for a specific wallet and mint combination. Ensures one canonical token account per wallet per token.

### Bump Seed
An additional byte added to PDA derivation seeds to ensure the resulting address is off the Ed25519 curve (no private key exists).

### Deserialize
Converting raw bytes from an account's data field into a structured type.

### Discriminator
An 8-byte identifier at the start of Anchor account data that uniquely identifies the account type.

### Idempotent
An operation that produces the same result regardless of how many times it's executed. Important for handling transaction retries.

### Instruction
A single operation within a transaction. Contains a program ID, account list, and instruction data.

### Keypair
A public key and corresponding private key used for signing transactions.

### Mint
The account that defines a token's properties (supply, decimals, authorities). Each token type has one mint.

### Owner
The program authorized to modify an account's data. System-owned accounts can only be modified by the System Program.

### Seeds
Byte arrays used to derive PDAs. Combined with a program ID to create deterministic addresses.

### Serialize
Converting a structured type into bytes for storage in an account's data field.

### Signer
An account that has signed the transaction, proving authorization for operations that require it.

### Writable
An account marked as modifiable within a transaction. Non-writable accounts can be read but not changed.

## Infrastructure Terms

### DAS (Digital Asset Standard)
A standard API for querying NFTs and tokens, especially compressed NFTs. Implemented by RPC providers like Helius.

### Geyser Plugin
A validator extension that streams real-time account and transaction data for indexing purposes.

### Indexer
A service that processes blockchain data and stores it in a queryable format (database, search index).

### RPC Provider
A service that runs Solana nodes and provides RPC API access. Examples: Helius, QuickNode, Alchemy.

### Webhook
An HTTP callback triggered by on-chain events. Used for real-time notifications without polling.

### WebSocket
A protocol for real-time bidirectional communication. Used for account subscriptions and live updates.

---

## Security Terms

### Authority
An account with special permissions over a resource (mint authority, freeze authority, upgrade authority).

### Freeze
The ability to prevent token transfers. Controlled by the freeze authority on a mint.

### Multisig
A setup requiring multiple signatures to authorize an action. Used for security-critical operations.

### Revoke
Removing an authority permanently. Once revoked, the authority cannot be restored.

### Time Lock
A delay period before certain operations can be executed. Provides time for review and potential intervention.

### Upgrade Authority
The account authorized to upgrade a program. Can be a keypair, multisig, or revoked (making the program immutable).

---

## Token Terms

### Burn
Permanently destroying tokens by reducing supply. Removes tokens from circulation.

### Decimals
The number of decimal places for a token. SOL has 9 decimals; USDC has 6.

### Delegation
Granting another account permission to transfer tokens on your behalf, up to a specified amount.

### Freeze Authority
The account authorized to freeze token accounts, preventing transfers.

### Metadata
Off-chain information about a token or NFT (name, symbol, image, attributes). Usually stored on IPFS or Arweave.

### Mint Authority
The account authorized to create new tokens. Can be revoked for fixed-supply tokens.

### Supply
The total number of tokens in existence. Can be fixed (mint authority revoked) or variable.

### Token Account
An account that holds a specific token type for a specific owner. Contains balance and owner information.

### Token-2022 (Token Extensions)
An upgraded token program with additional features: transfer fees, hooks, confidential transfers, non-transferable tokens.

---

## NFT Terms

### Collection
A group of related NFTs with a verified collection address. Used for organizing and verifying NFT sets.

### Compressed NFT (cNFT)
NFTs stored in Merkle trees for cost-efficient minting. 1000x cheaper than traditional NFTs.

### Creator
An address listed as a creator of an NFT, with an associated share of royalties.

### Merkle Tree
A data structure that enables efficient verification of data membership. Used for compressed NFTs.

### Metaplex
The leading NFT standard and tooling ecosystem on Solana. Defines metadata standards and provides minting tools.

### Non-Fungible Token (NFT)
A unique digital asset with supply of 1 and 0 decimals. Represents ownership of digital or physical items.

### Royalties
Fees paid to creators on secondary sales. Enforced by marketplaces, not at the protocol level.

### Verified
An attribute indicating an address has signed to confirm its association with an NFT (creator, collection).

---

## DeFi Terms

### AMM (Automated Market Maker)
A decentralized exchange mechanism using liquidity pools and mathematical formulas for pricing.

### Liquidity Pool
A smart contract holding paired assets for trading. Liquidity providers deposit assets and earn fees.

### Oracle
A service providing external data (prices, randomness) to on-chain programs.

### Slippage
The difference between expected and actual price due to market movement or insufficient liquidity.

### Swap
Exchanging one token for another through a DEX or AMM.

### TVL (Total Value Locked)
The total value of assets deposited in a DeFi protocol. A common metric for protocol size.

### Yield
Return on deposited assets, usually expressed as APY (Annual Percentage Yield).
