# 4.5 Tokens: SPL Token Program

> *"Most fungible tokens on Solana â€” from USDC to meme coins â€” use the SPL Token or Token-2022 program."*

## Why This Matters

The SPL Token Program is one of the most important programs on Solana. It defines how tokens work â€” creating new tokens, minting, transferring, burning, and more. Understanding it deeply enables you to build DeFi protocols, NFT platforms, and any application involving digital assets.

### Key Concepts

| Concept | Description |
|---------|-------------|
| **Mint** | Defines a token type (like a template) |
| **Token Account** | Holds a balance of a specific mint |
| **Decimals** | Precision (6 = millionths, 9 = billionths) |
| **Supply** | Total tokens in existence |
| **Authority** | Who can mint/freeze/close |

### ERC-20 vs SPL Token

| Aspect | ERC-20 (Ethereum) | SPL Token (Solana) |
|--------|-------------------|-------------------|
| Storage | In contract mapping | Separate accounts |
| Deployment | Each token = new contract | One program, many mints |
| Balances | `balances[address]` | Token account data |
| Approvals | `allowances[owner][spender]` | Delegate field |
| Gas/Fees | Per-contract fees | Unified program fees |

### The Mint Account

```rust
pub struct Mint {
    /// Optional authority to mint new tokens
    pub mint_authority: COption<Pubkey>,
    /// Total supply of tokens
    pub supply: u64,
    /// Number of decimals (0-9)
    pub decimals: u8,
    /// Is this mint initialized?
    pub is_initialized: bool,
    /// Optional authority to freeze token accounts
    pub freeze_authority: COption<Pubkey>,
}
```

**Important Fields:**

| Field | Purpose | Common Values |
|-------|---------|---------------|
| `mint_authority` | Who can create new tokens | Your program's PDA |
| `supply` | Total tokens minted - burned | Starts at 0 |
| `decimals` | Decimal places | 6 (USDC), 9 (SOL), 0 (NFT) |
| `freeze_authority` | Who can freeze accounts | Often null for DeFi |

### Associated Token Accounts (ATAs)

ATAs are the standard way to create token accounts:

```
ATA derivation:
  seeds = [owner, token_program_id, mint]
  â†’ Deterministic address for any wallet + token combo

Benefits:
  - Anyone can calculate the address
  - No need to store or communicate addresses
  - Standard across all wallets and dApps
```

```typescript
// Finding an ATA address
import { getAssociatedTokenAddress } from "@solana/spl-token";

  mintPubkey,     // Which token
  ownerPubkey,    // Whose wallet
);
```

### Minting Tokens

```rust
use anchor_spl::token::{self, MintTo};

#[derive(Accounts)]
pub struct MintTokens<'info> {
    #[account(mut)]
    pub mint: Account<'info, Mint>,

    #[account(
        mut,
        associated_token::mint = mint,
        associated_token::authority = recipient,
    )]
    pub recipient_ata: Account<'info, TokenAccount>,

    /// CHECK: Recipient of tokens
    pub recipient: UncheckedAccount<'info>,

    /// CHECK: PDA with mint authority
    #[account(
        seeds = [b"mint-authority", mint.key().as_ref()],
        bump
    )]
    pub mint_authority: UncheckedAccount<'info>,

    pub token_program: Program<'info, Token>,
}

pub fn mint_tokens(ctx: Context<MintTokens>, amount: u64) -> Result<()> {
    let mint_key = ctx.accounts.mint.key();
    let seeds = &[
        b"mint-authority",
        mint_key.as_ref(),
        &[ctx.bumps.mint_authority],
    ];
    let signer_seeds = &[&seeds[..]];

    token::mint_to(
        CpiContext::new_with_signer(
            ctx.accounts.token_program.to_account_info(),
            MintTo {
                mint: ctx.accounts.mint.to_account_info(),
                to: ctx.accounts.recipient_ata.to_account_info(),
                authority: ctx.accounts.mint_authority.to_account_info(),
            },
            signer_seeds,
        ),
        amount,
    )?;

    msg!("Minted {} tokens to {}", amount, ctx.accounts.recipient.key());
    Ok(())
}
```

## Client-Side Token Operations

### TypeScript SDK

```typescript
import {
  createMint,
  getOrCreateAssociatedTokenAccount,
  mintTo,
  transfer,
  burn,
  getAccount,
  getMint,
} from "@solana/spl-token";
import { Connection, Keypair, PublicKey } from "@solana/web3.js";

async function tokenOperations() {
  const connection = new Connection("https://api.devnet.solana.com");
  const payer = Keypair.generate(); // Your wallet

  // Create a new token mint
  const mint = await createMint(
    connection,
    payer,           // Payer
    payer.publicKey, // Mint authority
    null,            // Freeze authority (null = none)
    6                // Decimals
  );
  console.log("Mint:", mint.toBase58());

  // Get or create token account
  const tokenAccount = await getOrCreateAssociatedTokenAccount(
    connection,
    payer,
    mint,
    payer.publicKey
  );
  console.log("Token Account:", tokenAccount.address.toBase58());

  // Mint tokens
  await mintTo(
    connection,
    payer,
    mint,
    tokenAccount.address,
    payer, // Mint authority
    1_000_000_000 // 1000 tokens (with 6 decimals)
  );

  // Check balance
  const info = await getAccount(connection, tokenAccount.address);
  console.log("Balance:", Number(info.amount) / 1e6);

  // Transfer to another account
  const recipient = Keypair.generate();
  const recipientAta = await getOrCreateAssociatedTokenAccount(
    connection,
    payer,
    mint,
    recipient.publicKey
  );

  await transfer(
    connection,
    payer,
    tokenAccount.address,
    recipientAta.address,
    payer,
    500_000_000 // 500 tokens
  );

  // Burn tokens
  await burn(
    connection,
    payer,
    tokenAccount.address,
    mint,
    payer,
    100_000_000 // 100 tokens
  );
}
```

## Common Patterns

### Pattern 1: Staking Token

```rust
pub fn stake(ctx: Context<Stake>, amount: u64) -> Result<()> {
    // Transfer tokens to stake pool
    token::transfer(
        CpiContext::new(
            ctx.accounts.token_program.to_account_info(),
            Transfer {
                from: ctx.accounts.user_token.to_account_info(),
                to: ctx.accounts.stake_pool.to_account_info(),
                authority: ctx.accounts.user.to_account_info(),
            },
        ),
        amount,
    )?;

    // Update stake record
    let stake_account = &mut ctx.accounts.stake_account;
    stake_account.amount += amount;
    stake_account.staked_at = Clock::get()?.unix_timestamp;

    Ok(())
}
```

### Pattern 2: Token with Fixed Supply

```rust
pub fn initialize_fixed_supply(ctx: Context<InitFixedSupply>, total_supply: u64) -> Result<()> {
    // Mint entire supply
    let seeds = &[b"authority", &[ctx.bumps.authority]];

    token::mint_to(
        CpiContext::new_with_signer(
            ctx.accounts.token_program.to_account_info(),
            MintTo {
                mint: ctx.accounts.mint.to_account_info(),
                to: ctx.accounts.treasury.to_account_info(),
                authority: ctx.accounts.authority.to_account_info(),
            },
            &[seeds],
        ),
        total_supply,
    )?;

    // Remove mint authority (fixed supply forever)
    token::set_authority(
        CpiContext::new_with_signer(
            ctx.accounts.token_program.to_account_info(),
            SetAuthority {
                account_or_mint: ctx.accounts.mint.to_account_info(),
                current_authority: ctx.accounts.authority.to_account_info(),
            },
            &[seeds],
        ),
        AuthorityType::MintTokens,
        None, // Remove authority
    )?;

    Ok(())
}
```

## Key Takeaways

1. **One program, many tokens** â€” SPL Token is the universal standard
2. **Mint defines the token** â€” decimals, supply, authorities
3. **Token accounts hold balances** â€” separate from wallet accounts
4. **ATAs are deterministic** â€” standard derivation for any wallet+token
5. **Authorities control actions** â€” mint, freeze, close
6. **Delegation enables approvals** â€” like ERC-20 approve/transferFrom
7. **Closing returns rent** â€” always close empty accounts
